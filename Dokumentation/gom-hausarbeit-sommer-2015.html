Gestaltung Online-Medien

Entwerfen und realisieren Sie das Frontend einer Website.

Fachschaft MI

Die neue Fachschaft MI benötig wieder eine Website. Mögliche Inhalte sind z.B.

  • Aufgaben / Angaben der Fachschaft
  • Forum
  • Tauschbörse für Bücher, Hilfe, …
  • Informationen / Hilfen zum Studium
  • …

Überlegen Sie, was Ihnen für Inhalte sinnvoll erscheinen.

Alternative Aufgabe

Entwerfen Sie für eine reale Person, Firma oder Institution eine Website. Wichtig dabei: Es müssen reale Inhalte und die Einwilligung der Firma bzw. Institution vorliegen.

Vorgehen

Für die gewählte Website ist ein modernes, zeitgemäßes Design zu entwickeln. Das Design muss sich an unterschiedliche Endgeräte anpassen. Das Design ist zunächst als Style-Tiles zu entwerfen und später mit HTML5 und CSS für aktuelle Browser als responsiven Frontend-Prototyp umzusetzen.

Inhalte

Entwicklen Sie realistische Inhalte (Text und Bild) oder nutzen reale Inhalte für die Website.

Entwürfe als Style-Tiles

Entwickeln Sie zunächst drei alternative Entwürfe als Style-Tiles.

Die Entwürfe können Sie mit Indesign, Illustrator, o.Ä., oder als HTML/CSS erstellen.

Konzentrieren Sie sich dabei auf die Elemente der Inhaltsseiten und fügen evtl. weitere Elemente wie Logo, Fließtext, Eingabefelder, Buttons hinzu.

Entwicklung von Modulen

Aus diesen Style-Tiles entwickeln Sie dann komplexere Module (z.B. Suche-Feld, Anzeige eines Covers, …), als Element-Kollagen, die Sie mit HTML/CSS umsetzen.

HTML-Prototypen

Aus diesen Modulen entwickeln Sie dann einen responsiven HTML-Prototypen. Wichtig: Entwerfen Sie zunächst für mobile Endgeräte und passen das Design für immer größere Endgeräte an. (Mobile first)

Besprechung der Entwürfe

Bitte nutzen Sie die Chance die Entwürfe in den Laboren mit dem Dozenten und Tutoren zu besprechen.

Umfang des Prototypen

Für alle Sites gilt:

Der Prototyp soll mindestens 4 verschiedene Typen von Inhaltsseiten in Variationen und die Startseite umfassen.

Unter anderem:

  • Übersichtsseite (mehrere Artikel, Texte, Produkte, Forumseinträge, etc). In Variationen mit viel und wenig Inhalten.
  • Inhaltsseiten (einzelne Artikel, Text, Produkt, Forumsbeitrag, etc). In mehren Variationen mit viel und wenig Inhalten.
  • Formular (Kontaktformular, Bestellformular, Eingabe ins Forum, …)
  • Seite mit längerem Text (z.B. Über uns, o.Ä.)
  • Impressum
  • Startseite

Anforderungen an HTML, CSS usw.

Die Website ist mit HTML5 und CSS 3 umsetzen.

  • HTML und CSS müssen valide sein.
  • Semantisches HTML.
  • Mobile First.
  • Anpassung an verschieden Endgeräte über Mediaqueries.
  • Klare Struktur im CSS (SMACSS / BEM).
  • Klare Trennung von Struktur, Präsentation, Verhalten.

Die Website wird in aktuellen Versionen von Chrome, Firefox und Safari getestet. Auf veraltet Browser brauchen Sie keine Rücksicht zu nehmen.

Sie sollten einen Preprozessor für CSS (SCSS/LESS) verwenden. Sie müssen allerdings Quellcode und kompiliertes CSS abgeben.

Browserspezifische CSS-Erweiterungen (-mox-…, -webkit-…, -o-…) dürfen Sie verwenden – allerdings muss die Website auch ohne diese Erweiterungen lesbar/verwendbar sein. Verwenden Sie jeweils alle sinnvollen Prefixe – nicht nur einen.

Zusätzliche Web-Schriften – eingebunden über Typekit oder als freie Webfonts – sollten Sie verwenden. Bei der Nutzung von Typekit müssen Sie die Website zusätzlich auf einem externen Server (evtl. Passwort geschützt) zur Verfügung stellen.

Sie dürfen gerne zusätzliche Interaktionen mit Javascript realisieren – die Informationen und Inhalte müssen allerdings auch ohne Javascript zugänglich sein.

Severseitige Programmierung ist nicht nötig: Webshop, Forum etc müssen ohne Funktion sein.

Bewertung mit 5,0

Für folgende Sachverhalte wird die Arbeit mit 5,0 bewertet:

  • Keine ausreichende Auseinandersetzung mit der Gestaltung
  • Gestaltung ohne realistische Inhalte
  • Tabellen für Layout (auch für Formulare)
  • Inline-Styles
  • Inline Javascript (z.B. onclick="call_js("No!");")
  • Site ohne Javascript nur eingeschränkt zu nutzen
  • Viele Validationsfehler
  • Nicht-semantisches HTML (z.B. <div class="ueberschrift"> statt <h1>)
  • Nicht responsives Webdesign

Dokumentation

Folgende Fragen sollten in der Dokumentation behandeln:

  • Idee und Grafisches Konzept (keine Bildbeschreibung, sondern Beschreibung der gestalterischen Motivation)
  • Styletiles
  • Auswahl des Styletiles mit Begründung
  • Veränderungen gegenüber dem ersten Entwurfsschritt mit Begründung
  • Überlegungen zu Typografie, Farben, Layout, Raster (Warum wurde so gestaltet?)
  • Angaben zu Typografie, Farben, Layout, Raster (Werte und Beispiele)
  • Überlegungen zum Responsive Webdesign und dessen Umsetzung.
  • Erklärung, dass sie über die nötigen Bildrechte verfügen, ggf. mit URL der Bilder. Keine Ausdrucke der AGBs beifügen!

Zeitleiste

Dokumentieren Sie den Entwurfsprozess als Projekt-Zeitleiste analog zu Daniel Malls Project Hub.

Dabei soll die Zeit von oben nach unten fortschreiten, das Ergebnis also am Ende. Bauen Sie kleine Screenshots – verlinkt mit der Original-Datei – ein, nicht nur Links zu Dateien.

Form der Abgabe

Die Abgabe erfolgt (fast) rein digital:

  • Die Dokumentation ist im Design der Site ebenfalls in HTML und CSS zu erstellen (Nutzen Sie Markdown zur Erzeugung der Texte).
  • Ebenso ist die Zeitleiste in HTML und CSS zu erstellen.
  • Die Projekt-Zeitleiste und die Dokumentation sollen von der Site verlinkt sein.
  • Ihr Name, Loginname und Matrikelnummer muss in der Dokumentation zu finden sein.
  • Der Ordner-Name muss Ihren Loginnamen enthalten.

Benennen Sie den Ordner mit GOM-HA-[Loginname]. In dem Unterordner dokumentation sollen Dokumentation und Projekt-Zeitleiste zu finden sein. Den komprimierten Ordner laden Sie bitte auf StudIP hoch – oder senden mir einen Link zum Download (Nur eigenen Webspace, OwnCloud, Sync oder Dropbox).

Lediglich die Erklärung zu den Nutzungsrechten ist unterschrieben in Papierform abzugeben. (Postfach #5)

Bewertungskriterien

Ohne Anspruch auf Vollständigkeit:

  • Gestalterische Idee
  • Alternative Entwürfe
  • Gestalterische Qualität: Typografie, Farben, Layout, Raster, Bilder
  • Begründung der Gestaltung (Typografie, Farben, Layout, Raster, Bilder)
  • Valides HTML5, CSS
  • Semantisches Markup
  • Organisation CSS bzw. SCSS
  • Mobile First
  • Responsive Webdesign: Umfang, Aufwand, Qualität der Anpassungen an unterschiedliche Endgeräte
  • Trennung von Struktur, Präsentation, Verhalten
  • Evtl. zusätzliche Interaktion mittels Javascript
  • Umfang
  • Dokumentation des Entwurfsprozesses
  • Qualität der Dokumentation
